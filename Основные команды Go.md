## Основные команды языка Go и их применение
В Go есть несколько встроенных команд, которые облегчают разработку, запуск и сборку программ. Рассмотрим основные из них.
 
Первая команда — go run. Она используется для быстрого запуска программы. Команда компилирует исходный код (обычно из одного или нескольких файлов) во временный исполняемый файл и сразу же запускает его, не сохраняя созданный бинарный файл на диск. Это очень удобно, когда вы хотите мгновенно проверить, как работает ваш код. Например, если у вас есть файл с именем main.go, вы можете набрать в командной строке «go run main.go», и программа сразу выполнится.
 
Следующая команда — go build. Эта команда компилирует программу и создаёт постоянный исполняемый файл (бинарник). Такой файл можно запустить повторно, передать другому пользователю или развернуть на сервере. По умолчанию имя файла соответствует имени папки, в которой находится код, но вы можете изменить его с помощью флага -o. Например, команда «go build -o myapp ./cmd/server» означает следующее:
- go build — команда сборки программы
- -o myapp — флаг, который указывает имя выходного файла (в данном случае бинарник будет называться «myapp»)
- ./cmd/server — путь к каталогу, где находится исходный код для сборки.
Этот вариант полезен, если вы хотите иметь чёткое имя для вашего приложения, вместо автоматического назначения имени
 
Еще одна полезная команда — go fmt. Она автоматически форматирует ваш исходный код согласно стандартам, принятым в сообществе Go. Благодаря этому код становится более читаемым и единообразным. Очень часто программисты запускают «go fmt ./...», чтобы привести весь проект к единому стилю, особенно перед коммитом в систему контроля версий.
 
Команда go test запускает модульные тесты. В проектах Go тесты пишутся в файлах, оканчивающихся на _test.go. Используя go test, вы можете убедиться, что функции и методы вашего кода работают правильно. Например, «go test -v» запускает тесты с подробным выводом, что помогает в отладке. Наконец, рассмотрим команды, связанные с управлением зависимостями: go mod init и go mod tidy.
 
В современном программировании очень важно управлять зависимостями — внешними библиотеками, которые использует ваш проект. В Go для этой цели используются модули, которые описываются с помощью файла go.mod. Модули — это наборы пакетов, объединённые общим файлом конфигурации (go.mod), который управляет зависимостями и версиями. Такой модуль можно легко использовать, поддерживать и распространять.
 
Когда вы начинаете новый проект, первым шагом является инициализация модуля с помощью команды go mod init. Например, если вы набираете «go mod init myproject», это означает следующее:
- Команда создаёт файл go.mod в корневой директории вашего проекта
- В файле go.mod будет указано имя модуля (в данном случае «myproject»), которое часто соответствует пути до репозитория (например,github.com/username/myproject)
- Кроме того, этот файл будет содержать информацию о минимальной версии Go, которую использует проект
 
В процессе разработки вы, скорее всего, будете добавлять или удалять зависимости, то есть подключать библиотеки, которые упрощают выполнение задач в вашем коде. Иногда зависимости добавляются автоматически, когда вы импортируете нужные пакеты, а иногда — вручную, с помощью команды go get.
 
Чтобы поддерживать файл go.mod (а также сопутствующий файл go.sum, который содержит контрольные суммы для подтверждения целостности библиотек) в актуальном состоянии, используется команда go mod tidy. Команда go mod tidy выполняет следующие действия:
- Проходит по вашему исходному коду и определяет, какие зависимости на самом деле используются
- Удаляет записи в go.mod о тех библиотеках, которые больше не нужны
- Добавляет в файл go.mod те зависимости, которые используются, но по какимто причинам не были указаны, например, если были добавлены через go get
- Обновляет файл go.sum, чтобы контрольные суммы соответствовали актуальным версиям библиотек
 
Таким образом, go mod tidy помогает поддерживать минимальный, чистый и актуальный список зависимостей в проекте, что особенно важно для стабильности сборки и облегчения работы команды разработчиков.

## Особенности синтаксиса языка Go
Изучение любого языка программирования начинается с понимания его базового синтаксиса — тех основ, на которых строится вся логика будущих приложений. Go (или Golang) — строготипизированный, компилируемый язык с лаконичным синтаксисом, ориентированным на простоту, читаемость и производительность. Ниже мы рассмотрим, из чего состоит типичная программа на Go и какие правила
важно соблюдать при её написании.
 
**Объявление пакета: package main**
Любая программа на Go начинается с объявления пакета. Пакеты (packages) — это базовая единица организации кода в Go. Для исполняемых программ, то есть таких, которые должны запускаться напрямую, используется специальный пакет main. Это своего рода сигнал компилятору: эта программа должна компилироваться как исполняемый файл, а не как библиотека.
```go
package main
```
Если вы пишете библиотеку (например, набор функций, которые потом будут использоваться в других программах), вы можете указать другое имя пакета. Но в случае самостоятельной программы, без package main ничего не заработает.
 
**Импорт пакетов: import**
Далее следует блок импорта. Здесь вы подключаете внешние и стандартные библиотеки, которые собираетесь использовать в своём коде. Например, чтобы вывести текст в консоль, нужно использовать пакет fmt, который входит в стандартную библиотеку Go:
```go
import "fmt"
```
Если вы планируете использовать сразу несколько пакетов, их можно перечислить в блочном формате — внутри круглых скобок:
```go
import (
    "fmt"
    // другие пакеты по мере необходимости
)
```
Комментарии в Go начинаются с двойного слэша // и могут быть размещены после строки кода или отдельно

**Точка входа в программу: func main()**
После импорта следует ключевая часть любой исполняемой программы на Go — функция main. Именно с неё начинается выполнение вашего приложения.
```go
func main() {
    // код, который будет выполнен при запуске программы
}
```
Всё, что вы поместите внутрь фигурных скобок { }, будет выполняться при запуске программы. Это место, где разворачивается основная логика: обработка данных, вызовы других функций, взаимодействие с пользователем и т.д.
 
**Переменные и короткое объявление: :=**
Go поддерживает как традиционный способ объявления переменных с указанием типа, так и упрощённый — с помощью оператора :=, когда тип выводится автоматически:
```go
a := 1
```
Здесь мы создаём переменную a, присваиваем ей значение 1, и компилятор сам понимает, что это int (целое число). Это делает код короче и чище, особенно в простых сценариях.

**Условные конструкции: if, return**
Условные операторы используются для проверки логики и принятия решений в коде. Один из самых распространённых — if. Он помогает определить, нужно ли выполнить тот или иной участок кода.
 
Представим, что вы пишете программу, в которой необходимо разделить одно число на другое. Но перед этим важно убедиться, что делитель не равен нулю — иначе произойдёт ошибка. Для этого используется if:
```go
if b == 0 {
    fmt.Println("Ошибка: деление на ноль")
    return
}
```
Здесь программа проверяет: если b равен нулю, выводит сообщение об ошибке и завершает выполнение функции с помощью return.

**Пример 1: простейший вывод в консоль**
```go
package main

import (
    "fmt"
)

func main() {
    fmt.Println("Hello, Go!") // выводим строку в консоль
}
```

## Различия между командами go run и go build
Команды go run и go build работают с вашим Go‑кодом, но предназначены для разных целей. Команда go run компилирует и сразу запускает код, что удобно для быстрого тестирования и отладки, но она не создаёт постоянный исполняемый файл на диске. Это значит, что если вы захотите запустить программу повторно, вам снова придётся выполнять go run.
 
В противоположность этому, команда go build создаёт исполняемый бинарный файл, который сохраняется в вашей файловой системе. Такой файл можно запускать многократно, передавать другим пользователям или разворачивать в продакшене. Как уже упоминалось, если вы хотите, чтобы бинарный файл имел определённое
имя, можно использовать флаг -o. Например, команда go build -o myapp ./cmd/server скомпилирует код, находящийся в папке cmd/server, и создаст файл с именем myapp. Флаг -o говорит компилятору, что нужно записать результат сборки в файл с указанным именем (в данном случае «myapp»).
