# Фунции (functions)
Функция — это блок кода, который выполняет определённую задачу. Она принимает входные данные (аргументы), обрабатывает их и возвращает результат. Представьте себе рецепт приготовления блюда. Он описывает, какие ингредиенты нужны и какие шаги нужно выполнить, чтобы получить готовое блюдо. Функция в Go —  как этот рецепт.
 
**Зачем нужны функции:**
- Модульность. Функции разбивают большие программы на более мелкие, управляемые части.
- Переиспользование кода. Функции позволяют повторно использовать код в разных частях программы, избегая дублирования.
- Читаемость. Функции делают код более читаемым и понятным, поскольку каждая функция выполняет чётко определённую задачу.
- Тестируемость. Функции можно тестировать независимо друг от друга, что упрощает процесс отладки и проверки кода.

**Как создать функцию**
Используйте ключевое слово func, за которым следует имя функции, список аргументов (если есть), список типов возвращаемых значений (если есть) и тело функции.
```go
func Add(x int, y int) int {
    return x + y
}
```
**Как использовать функцию**
Чтобы использовать функцию, просто вызовите её по имени, передав необходимые аргументы.
```go
// main.go
package main

import "fmt"

// Функция, которая складывает два целых числа
func Add (x int, y int) int {
    return x + Y
}

func main () {
    sum := Add (5, 3)
    fmt. Println ("Сумма:", sum) // Выведет: Сумма: 8
}
```

## Лучшие практики использования функций
1. Именование функций:
- Используйте осмысленные имена. Имя функции должно отражать её функциональность. Например, CalculateArea, ValidateInput, SendEmail.
- Используйте глаголы. Имена функций часто начинаются с глагола, чтобы указать, что они делают. Например, Get, Set, Calculate.
- Следуйте конвенциям. В Go принято использовать CamelCase для имён функций.
2. Аргументы:
- Ограничивайте количество аргументов. Функции с большим количеством аргументов сложны для понимания и использования. По возможности используйте структуры для группировки связанных аргументов.
- Используйте указатели для изменяемых аргументов. Если функция должна изменить значение аргумента, передавайте его как указатель.
3. Возвращаемые значения:
- Возвращайте несколько значений. Go позволяет функциям возвращать несколько значений. Это удобно для возврата результата и ошибки одновременно.
- Используйте именованные возвращаемые значения. Именованные возвращаемые значения делают код более читаемым и понятным. Пример:
```go
func CarCoordinates(carID int) (lat float32, lon float32, err error)
```
4. Обработка ошибок:
- Используйте error-интерфейс. В Go используется стандартный интерфейс error для представления ошибок.
- Проверяйте ошибки. Если функция может вернуть ошибку, обязательно проверяйте её и обрабатывайте соответствующим образом.
```go
lat, lon, err := CarCoordinates(carID)
if err != nil {
    fmt.Printf("Ошибка получения координат по автомобилю (id=%d): %w\n, carID, err")
}
```
5. Тестирование:
- Пишите тесты для каждой функции. Тесты помогают убедиться, что ваша функция работает правильно и изменения не приводят к поломкам.
6. Документирование:
- Пишите комментарии для каждой функции. Комментарии помогают другим разработчикам понять, как использовать вашу функцию. Используйте стандартные комментарии Go для документирования параметров, возвращаемых значений и возможных ошибок. Начинаться комментарий должен с имени функции.

# Пакеты (packages)
Пакет — это коллекция связанных файлов исходного кода, объединённых под общим именем. По сути, это способ организации вашего кода в логические блоки. Представьте себе, что вы строите дом. У вас есть фундамент, стены, крыша, окна. Каждый из этих элементов — это отдельный модуль, который выполняет свою функцию. Пакеты в Go — это как эти модули, только для вашего кода.
 
**Зачем нужны пакеты:**
- Организация кода: пакеты помогают структурировать большие проекты, разбивая их на более мелкие, управляемые части.
- Переиспользование кода: пакеты позволяют вам повторно использовать код в разных частях вашего проекта или даже в других проектах.
- Инкапсуляция: пакеты позволяют скрыть детали реализации от внешнего мира, предоставляя только необходимый интерфейс. Это способствует модульности и упрощает внесение изменений в код.
- Управление зависимостями: Go имеет встроенную систему управления зависимостями, которая позволяет легко использовать пакеты, разработанные другими разработчиками.

**Как создать пакет**
- Для объявления пакета в языке Go используется ключевое слово package. Вам нужно создать папку с именем пакета и поместить в неё ваши файлы .go.
- Например, если вы хотите создать пакет с именем geometry, вы создаёте папку geometry и помещаете в неё файлы, содержащие функции и типы, связанные с геометрией.
```go
// geometry/rectangle.go
package geometry

type Rectangle struct {
    Width float64
    Height float64
}

func (r *Rectangle) Area () float64 {
    return r. Width * r. Height
}
```
**Как использовать пакет**
Чтобы использовать пакет в другом файле, вам нужно импортировать его с помощью ключевого слова import.
```go
// main. go
package main

import "example.com/myproject/geometry" // Замените на свой путь к пакету

func main () {
    rect := geometry.Rectangle{Width: 10, Height: 5}
    area : = rect.Area ()
    printin ("Площадь прямоугольника:", area)
}
```
**Лучшие практики использования пакетов**
Теперь перейдём к важной части — лучшим практикам. Правильное использование пакетов — это искусство, которое требует внимания к деталям.

1. Именование пакетов:
- Используйте осмысленные имена. Имя пакета должно отражать его функциональность. Например: database, http, auth.
- Используйте обратную нотацию доменных имён. Это помогает избежать конфликтов имён с другими пакетами. Например: github.com/yourusername/yourproject/database.
- Избегайте общих имён. Не используйте такие имена, как utils или helpers. Подумайте, что именно делает пакет, и дайте ему более конкретное имя.
- Начните с маленькой буквы. В Go идентификаторы (переменные, функции, типы), начинающиеся с заглавной буквы, являются публичными (exported) и доступны извне пакета. Идентификаторы, начинающиеся с маленькой буквы, являются приватными и доступны только внутри пакета.
- Экспортируйте только необходимое. Не экспортируйте всё, что у вас есть в пакете. Экспортируйте только те части, которые должны быть доступны извне. Это помогает скрыть детали реализации и упрощает внесение изменений в код.
2. Зависимости:
- Минимизируйте зависимости. Чем меньше зависимостей у пакета, тем проще его поддерживать и тестировать.
- Явно указывайте зависимости. В файле go.mod явно указывайте все зависимости вашего проекта.
3. Тестирование:
- Пишите тесты для каждого пакета. Тесты помогают убедиться, что ваш код работает правильно и что изменения не приводят к поломкам. Имя тестового файла должно быть таким же, как и имя пакета, с добавлением суффикса _test.go.
4. Документирование:
- Пишите комментарии. Комментарии помогают другим разработчикам понять, как использовать ваш пакет. Используйте стандартные комментарии Go для документирования экспортированных функций, типов и переменных. Комментарий для пакета должен начинаться со слов Package <имя пакета>.

# Пакет fmt
Пакет fmt (сокращение от «format») — один из основных пакетов в Go, который используется для форматированного ввода-вывода данных.

Основные функции пакета fmt
1. `fmt.Println()` - Описание: Выводит аргументы на экран и добавляет символ перевода строки в конце.
```go
fmt.Println("=== Информация о товаре ===")
fmt.Println("\nДополнительная информация:")
```
Особенности:
- автоматически вставляет пробел между аргументами;
- автоматически добавляет символ новой строки в конце;
- принимает любое количество аргументов любого типа.
2. `fmt.Printf()` - Описание: Форматированный вывод, позволяющий контролировать формат представления данных.
```go
fmt.Ptintf("ID: %d\n, id")
fmt.Printf("Цена: %.2f руб.\n", price)
```
Особенности:
- первый аргумент — строка форматирования с параметрами подстановки;
- последующие аргументы — значения для подстановки;
- не добавляет автоматически символ новой строки (нужно указывать `\n`).

**Параметры форматирования (спецификаторы) fmt.Printf()**
 
В функции `fmt.Printf()` используются специальные последовательности, начинающиеся с символа `%`, которые определяют, как форматировать различные типы данных.
 
**Для целых чисел:**
```go
fmt.Printf("ID: %d\n", id) // ID: 12345
```
- `%d` — десятичное целое число;
- `%b` — двоичное представление;
- `%o` — восьмеричное представление;
- `%x` или `%X` — шестнадцатеричное представление (строчными или заглавными буквами).
 
**Для чисел с плавающей точкой:**
```go
fmt.Printf("Вес товара: %.2f кг\n", itemWeight) // Вес товара: 0.30 кг
```
- `%f` — число с плавающей точкой;
- `%.2f` — число с плавающей точкой с указанной точностью (2 знака после запятой);
- `%e` или `%E` — экспоненциальная запись (например, 1.2e+9).
 
**Для строк:**
```go
fmt.Printf("Название: %s\n", name) // Название: Смартфон
```
- `%s` — строковое значение. 
 
**Для логических значений:**
```go
fmt.Printf("Доступно: %t\n", isAvailable) // Доступно: true
```
`%t` — логическое значение (true или false).
 
**Для произвольных значений:**
```go
fmt.Printf("complex64: %v\n", complexValue) // complex64: (1+2i)
```
`%v` — представление значения в формате по умолчанию;
`%#v` — представление значения в синтаксисе Go;
`%T` — тип значения.
 
**Дополнительные модификаторы:**
```go
fmt.Printf("Процент от максимального количества на складе: %.1f%%\n", percentInStock) // Процент от максимального количества на складе: 15.0%
```
- `%` — для печати символа процента: `%%`.

## Ещё полезные функции пакета fmt
`fmt.Sprintf()` — работает как `Printf()`, но вместо вывода на экран возвращает отформатированную строку.
