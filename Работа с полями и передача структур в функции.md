# Работа с полями структуры
### Доступ к полям структуры
После объявления структуры и создания её экземпляра вы можете обращаться к отдельным полям структуры с помощью оператора точки .. Это основной способ получения доступа к данным, хранящимся внутри структуры.
 
Допустим, у нас есть такая структура:
```go
type User struct {
    Name string
    Age int
}
```
Чтобы создать переменную типа User и получить доступ к её полям, нужно написать:
```go
var u User
u.Name = "Иван"
u.Age = 30

fmt.Println(u.Name)
fmt.Println(u.Age)
```
Оператор . позволяет не только читать значения полей, но и записывать новые. Это будет показано далее.
 
Важно помнить, что вы можете использовать точечную нотацию на любом экземпляре структуры: переменной, возвращаемом значении, результатах вызова функции и даже в цепочке выражений.
```go
fmt.Println(getUser().Name)
```
### Инициализация полей при создании структуры
При создании переменной структуры вы можете сразу задать значения её полей. Go предлагает несколько способов инициализации структур:
 
1. Инициализация с использованием позиционного литерала.
```go
type Point struct {
    X int
    Y int
}

p := Point{10, 20}
fmt.Println(p.X) // 10
fmt.Println(p.Y) // 20
```
Здесь значения полей задаются по порядку, в точном соответствии с тем, как они определены в структуре. Такой способ подходит для простых структур, где порядок очевиден. Однако если структура имеет много полей, использование позиционной инициализации может привести к ошибкам, особенно при изменении порядка полей в будущем.
  
2. Инициализация с использованием именованных полей.

Рекомендуется использовать именно этот способ — он делает код более читаемым:
```go
p := Point{
    Y: 20,
    X: 10,
}
```
Порядок указания полей не имеет значения, главное — правильно указать имена. Такой подход безопаснее и понятнее.

3. Инициализация структуры по умолчанию.

Если вы создаёте структуру без явной инициализации, все поля получают нулевые значения соответствующих типов:
```go
var u User
fmt.Println(u.Name) // пустая строка ""
fmt.Println(u.Age) // 0
```
Для строк это будет "", для чисел — 0, для булевых — false, для указателей и срезов — nil.
 
4. Инициализация структуры через new.
 
Можно создать структуру с помощью функции new, которая возвращает указатель на нулевое значение типа:
```go
u := new(User)
iu.Name = "Анна"
fmt.Println(u.Name)
```
 
**Изменение значений полей**
 
Как уже было показано, поля структуры можно свободно изменять после создания экземпляра. Для этого также используется оператор .:
```go
type Product struct {
    Name string
    Price float64
func main () {
    var p Product
    p.Name = "Ноутбук"
    p.Price = 54999.90
    fmt. Println (p.Name) // Ноутбук 
    fmt. Println(p.Price) // 54999.9
    // Изменим цену:
    p.Price = 49999.90
    fmt.Printin(p.Price) // 49999.9
}
```
**Обратите внимание:**
 
Если структура передаётся как значение, то вы работаете с копией, и изменения не будут видны вне функции. Но об этом мы поговорим позже в разделе передачи структур. Также следует помнить, что вы не можете менять значения неэкспортируемых полей извне пакета, если имя поля начинается с маленькой буквы.

## Подсказки по стилю и практике
- Используйте именованные литералы при инициализации структур: это делает код более устойчивым и понятным.
- Инициализируйте структуру сразу, если известно начальное состояние.
- Комментируйте поля, особенно если их назначение не очевидно.
- Разграничивайте экспортируемые и неэкспортируемые поля: если вы проектируете библиотеку или API, важно контролировать, какие поля доступны пользователям.
- Используйте функции-конструкторы, если структура имеет сложные зависимости. Это повышает читаемость и защищает от ошибок.
 
Пример функции-конструктора:
```go
func NewUser(name string, age int) User {
    return User{
        Name: name,
        Age: age,
    }
}
```
# Передача структур в функции
Работа с функциями — неотъемлемая часть любого языка программирования. В Go, как и в большинстве языков, функции могут принимать параметры и возвращать значения. Структуры, как сложные типы данных, тоже могут быть переданы в функции — как аргументы или как возвращаемые значения.
 
Важно понимать, как именно передаются структуры в функции: по значению или по ссылке. Это влияет на поведение программы, производительность и управление памятью. В этом разделе мы подробно разберём:
- как передавать структуры в функции;
- в чём разница между передачей по значению и по ссылке;
- как правильно выбирать способ передачи;
- и, конечно, увидим всё это на наглядных примерах.

### Как передавать структуры в функции
В Go передача аргументов в функции по умолчанию осуществляется по значению. Это означает, что при вызове функции создаётся копия передаваемого значения. Это правило распространяется и на структуры.
 
Однако Go также позволяет передавать указатели на структуры, что фактически даёт поведение «передачи по ссылке».
- Если структура передаётся как значение (MyStruct), то функция получает копию.
- Если структура передаётся как указатель (*MyStruct), то функция получает доступ к оригинальному объекту в памяти.
 
**Передача по значению**
 
При передаче структуры по значению создаётся копия всех её полей. Это безопасный, но потенциально затратный по производительности способ, особенно если структура большая.
```go
type User struct {
    Name string
    Age int
func updateAge (u User) {
    u. Age = u. Age + 1
    fmt. Printin ("Внутри функции:", u.Age)
}
func main () {
    user := User{Name: "Анна", Age: 30}
    updateAge (user)
    fmt. Printin ("Снаружи функции:", user. Age)
}
```
**Вывод:**
 
Внутри функции: 31
 
Снаружи функции: 30
 
Что произошло? Несмотря на то, что в функции updateAge мы увеличили возраст, это изменение затронуло только копию структуры. Оригинальная переменная user осталась неизменной.
 
Этот подход безопасен: мы не можем случайно изменить исходную структуру. Однако при передаче больших структур (например, с десятками полей или вложенными данными) это может негативно сказаться на производительности из-за копирования данных.
 
**Передача по ссылке (через указатель)**
 
Чтобы изменить оригинальную структуру внутри функции, её нужно передавать по ссылке — то есть передавать указатель.

**Как Go работает с указателями**
 
В Go указатель на тип обозначается звёздочкой (*). Например:
- *User — указатель на структуру User.
- &user — взятие адреса переменной user.
 
Когда вы передаёте &user в функцию, вы фактически передаёте адрес в памяти, по которому находится структура.

Как Go работает с указателями

В Go указатель на тип обозначается звёздочкой (*). Например:
*User — указатель на структуру User.
&user — взятие адреса переменной user.
Когда вы передаёте &user в функцию, вы фактически передаёте адрес в памяти, по которому находится структура.

**Сравнение: по значению vs. по ссылке**
 
|Критерий|Передача по значению|Передача по ссылке|
|--------|--------------------|------------------|
|Что передаётся|Копия структуры|Адрес оригинальной структуры|
|Изменения сохраняются?|Нет|Да|
|Безопасность|Высокая|Ниже (возможны нежелательные изменения)|
|Использование памяти|Может быть затратным|Эффективнее для больших структур|
|Применение|Когда нужны копии|Когда нужно изменить оригинал|

**Когда использовать какой подход**
- Передача по значению:
    - структура маленькая (например, 1–2 поля);
    - изменения внутри функции не должны влиять на оригинал;
    - нужна изоляция данных.
- Передача по ссылке (указатель):
    - структура большая;
    - требуется изменить структуру в вызывающем коде;
    - часто вызывается в цикле или в функции, чувствительной к производительности.
