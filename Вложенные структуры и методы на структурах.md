# Вложенные структуры в Go

Вложенная структура — это структура, которая используется как поле внутри другой структуры.

Рассмотрим простой пример. Представим, что у нас есть структура Address, которая описывает адрес:
```go
type Address struct {
    City string
    ZipCode string
}
```
Теперь создадим структуру User, у которой есть поле типа Address:
```go
type User struct {
    Name string
    Age int
    Address Address
}
```
Здесь структура Address вложена внутрь структуры User. Это означает, что каждый пользователь (User) будет иметь свой собственный адрес (Address) с городом и индексом.
 
Вложенные структуры позволяют логически группировать данные. Это делает структуру более наглядной, особенно в случае, когда объект содержит сложную или составную информацию.

### Синтаксис объявления вложенных структур
Объявление вложенной структуры ничем не отличается от добавления обычного поля. В качестве типа поля указывается другая структура.
```go
type ContactInfo struct {
    Email string
    Phone string
}

type Employee struct {
    Name string
    Department string
    Contact ContactInfo
}
```
Здесь структура ContactInfo вложена в структуру Employee.
 
Также в Go поддерживается анонимное встраивание структур, при котором структура указывается без имени поля. Например:
```go
type Person struct {
    Name string
    Age int
}

type Employee struct {
    Person  // анонимное встраивание
    Position string
    Department string
}
```
В этом случае поля структуры Person становятся доступными напрямую через Employee, как будто они объявлены внутри Employee. Это особенность Go и важный инструмент при построении гибких структур данных. Однако в этой главе мы будем использовать только именованные вложенные структуры, чтобы сохранить ясность.

### Доступ к полям вложенных структур

Чтобы обратиться к полям вложенной структуры, используется обычная точечная нотация (.), но с указанием вложенности.

Продолжим с предыдущего примера:
```go
type Address struct {
    City string
    ZipCode string
}

type User struct {
    Name string
    Address Address
}
```
Объявим переменную:
```go
user: = User{
    Name: "Ольга",
    Address: Address{
        City: "Москва",
        ZipCode: "101000",
    },
}
```
Теперь, чтобы получить доступ к городу пользователя, мы пишем:
```go
fmt.Println(user.Address.City) // Выведет: Москва
```
Аналогично, чтобы изменить поле вложенной структуры:
```go
user.Address.ZipCode = "105005"
```
Если вложенность глубокая, обращение к полям будет выглядеть как цепочка:
```go
user.Profile.Contact.Email
```
Чем глубже структура, тем длиннее путь к нужному полю. Это необходимо учитывать при проектировании моделей данных.

### Плюсы и минусы вложенные структуры
**Плюсы:**
- Логическая группировка данных. Упрощает понимание структуры объекта.
- Повторное использование кода. Один и тот же тип можно использовать в разных структурах.
- Модульность. Изменения в одной структуре не влияют напрямую на другие.
- Удобство при сериализации/десериализации. Структуры можно легко преобразовать в JSON и обратно.
**Минусы:**
- Сложность при глубокой вложенности. Могут быть длинные обращения к полям (obj.A.B.C.D).
- Затруднённая отладка. Особенно при больших структурах и использовании указателей.

# Методы на структурах в Go
В языке Go структуры позволяют объединять данные, описывающие какой-либо объект, например, пользователя, книгу или заказ. Но одних только данных часто бывает недостаточно. Нам также нужно описывать поведение объектов: что они могут делать, как могут изменяться, какие действия могут выполнять. Для этого в Go используются методы на структурах.
 
Методы позволяют "прикрепить" функцию к структуре, чтобы она могла выполнять определённые действия над её полями. Это приближает язык Go к объектно-ориентированному стилю, хотя Go не поддерживает классы в традиционном понимании. Вместо этого Go предлагает более простой, но мощный способ работы с методами через концепцию receiver (получателя).
 
Метод в Go — это функция, которая привязана к конкретному типу, чаще всего к структуре.

Он позволяет выполнять операции над экземплярами структуры и обращаться к их полям напрямую.

Таким образом, методы расширяют структуру действиями. Вместо того чтобы писать отдельные функции, которые принимают структуру как аргумент, мы можем сделать эти функции частью самой структуры.

Пример: если у нас есть структура User, мы можем определить метод Greet, который будет печатать приветствие от имени пользователя:
```go
type User struct {
    Name string
}

func (u User) Greet() {
    fmt.Println("Привет,", u.Name)
}
```
Теперь можно вызвать этот метод для любого значения типа User:
```go
user := User{Name: "Анна"}
user.Greet() // Привет, Анна
```

### Синтаксис объявления метода
Объявление метода похоже на объявление обычной функции, но с одним важным отличием — перед названием метода указывается receiver (получатель) в круглых скобках:
```go
func (r TypeName) MethodName() {
    // тело метода
}
```
Где:
- TypeName — имя типа (чаще всего структуры), к которому привязан метод;
- r — имя переменной-получателя, через которую можно обращаться к полям;
- MethodName — имя метода.
Receiver может быть либо значением (TypeName), либо указателем (*TypeName).
```go
type Rectangle struct {
    Width float64
    Heigt float64
}

func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}
```
Здесь метод Area возвращает площадь прямоугольника. Он получает значение типа Rectangle и работает с его полями Width и Height.

### Receiver: значение или указатель

Receiver (получатель) указывает на структуру, к которой относится метод. Он может быть двух видов:
- Значение (r Type) — метод получает копию структуры. Изменения в методе не затрагивают оригинальные данные.
- Указатель (r *Type) — метод получает ссылку на структуру. Изменения внутри метода сохраняются.
Рассмотрим оба варианта на примере:
```go
type Counter struct {
    Value int
}
// Метод с получателем-значением
func (c Counter) IncrementByValue() {
    c.Value++
}

/// Метод с получателем-указателем
func (c *Counter) IncrementByPointer() {
    c.Value++
}
```
Теперь протестируем:
```go
func main() {
    c := Counter{Value: 10}

    c.IncrementByValue()
    fmt.Println(c.Value) // 10 - значение не изменилось

    c.IncrementByPointer()
    fmt.Println(c.Value) // 11 - значение изменилось

}
```
Если вы хотите, чтобы метод модифицировал поля структуры, всегда используйте указатель в receiver.

### Для чего использовать методы на структурах
Методы — важный инструмент для построения инкапсулированной логики. Вместо того чтобы выносить поведение в отдельные функции, можно привязать его к самим структурам. Это делает код более читаемым, модульным и ближе к реальным моделям.
 
**Преимущества:**
- Поведение рядом с данными. Поведение логически связано с типом.
- Инкапсуляция. Можно скрыть детали реализации и предоставить только нужные методы.
- Переиспользуемость. Методы можно переиспользовать в различных частях программы.
- Работа с интерфейсами. Методы позволяют реализовывать интерфейсы Go.

### Метод vs функция

**Метод:**
```go
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}
```
**Функция:**
```go
func Area(r Rectangle) float64 {
    return r.Width * r.Height
}
```
Функция требует явной передачи аргумента, тогда как метод вызывается у объекта. Методы предпочтительнее, если они логически связаны с типом.
